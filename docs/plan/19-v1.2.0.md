# 19. Cambios de v1.2.0

## Resumen de v1.2.0

**v1.2.0** corrige problemas de UX identificados en testing y unifica el comportamiento entre tiradas automáticas y manuales:

1. **Guardado Consistente** - Botón de guardar siempre visible con valor por defecto
2. **Consultante Opcional** - Campo opcional en ambos modos (automático y manual)
3. **Carga Manual Mejorada** - Visualización con cartas de dorso + interpretación en misma pantalla
4. **Selector Visual** - Imágenes de cartas en CardSelectorScreen

---

## Cambios en UI/UX

### QuestionScreen (Reutilizada)

**Antes (v1.1.0)**:
- Modo automático: Toggle OFF por defecto
- Modo manual: Toggle ON forzado (no desactivable)
- Campo consultante obligatorio en modo manual

**Ahora (v1.2.0)**:
- **Ambos modos**: Toggle OFF por defecto
- **Ambos modos**: Campo consultante opcional
- Comportamiento idéntico entre modos (solo cambia hint visual)

```kotlin
// v1.2.0: Toggle OFF por defecto en AMBOS modos
var isForSomeoneElse by remember { mutableStateOf(false) }

// Sin diferencias de comportamiento por modo
Switch(
    checked = isForSomeoneElse,
    onCheckedChange = { isForSomeoneElse = it }
)
```

### ReadingScreen (Actualizada)

**Antes (v1.1.0)**:
- Botón "Guardar en Historial" solo visible si `consultantName != null`
- No se podía guardar si no se había ingresado consultante

**Ahora (v1.2.0)**:
- Botón "Guardar en Historial" **siempre visible**
- Usa valor por defecto `"Lectura personal"` si no hay consultantName

```kotlin
// v1.2.0: Siempre visible
val finalConsultantName = consultantName ?: "Lectura personal"

Button(
    onClick = { viewModel.saveToHistory(reading, interpretation, finalConsultantName) }
) {
    Text("Guardar en Historial")
}
```

### ManualLoadScreen (Rediseñada)

**Antes (v1.1.0)**:
- Posiciones mostraban "+" y nombre de posición
- Guardado automático al generar interpretación
- Navegaba a ReadingDetailScreen después de interpretar

**Ahora (v1.2.0)**:
- **Cartas de dorso** (imagen `card_back.jpg`) en posiciones
- Layout visual **idéntico a ReadingScreen**
- **Interpretación en misma pantalla** (scroll hacia abajo)
- **Guardado manual** con botón (igual que ReadingScreen)
- Navega a ReadingDetailScreen solo después de guardar

**Componentes reutilizados**:
- `HorizontalCardsLayout` - Para tiradas lineales
- `CrossCardsLayout` - Para tirada Cruz
- `GeneralInterpretationCard` - Para mostrar interpretación
- `YesNoAnswerCard` - Para tiradas Sí/No

**Estados de ManualLoadScreen**:

```kotlin
sealed class ManualLoadUiState {
    data object Loading : ManualLoadUiState()

    data class LoadingCards(
        val configuration: ManualLoadConfiguration

    data class InterpretationGenerated(
        val configuration: ManualLoadConfiguration,

    data class Error(val message: String) : ManualLoadUiState()

```

### CardSelectorScreen (Mejorada)

**Antes (v1.1.0)**:
- Grid mostraba solo nombres de cartas (texto)
- Difícil identificar cartas visualmente

**Ahora (v1.2.0)**:
- Grid muestra **imágenes de cartas** + nombre
- Usa `AsyncImage` (Coil) para cargar imágenes
- Layout idéntico a EncyclopediaScreen

```kotlin
// v1.2.0: CardItem con imagen
@Composable
fun CardSelectorItem(
    card: TarotCard,
    isSelected: Boolean,
    onCardClick: () -> Unit
) {
    Card(
        modifier = Modifier
            .clickable(onClick = onCardClick, enabled = !isSelected),
        colors = if (isSelected) {
            CardDefaults.cardColors(
                containerColor = MaterialTheme.colorScheme.surfaceVariant.copy(alpha = 0.5f)
            )
        } else {
            CardDefaults.cardColors()
        }
    ) {
        Column(
            modifier = Modifier.padding(8.dp),
            horizontalAlignment = Alignment.CenterHorizontally
        ) {
            AsyncImage(
                model = card.imageResId,
                contentDescription = card.name,
                modifier = Modifier
                    .fillMaxWidth()
                    .aspectRatio(0.7f),
                contentScale = ContentScale.Fit
            )

            Text(
                text = card.name,
                style = MaterialTheme.typography.bodySmall,
                textAlign = TextAlign.Center,
                maxLines = 2,
                overflow = TextOverflow.Ellipsis
            )

            if (isSelected) {
                Text(
                    text = "En uso",
                    style = MaterialTheme.typography.labelSmall,
                    color = MaterialTheme.colorScheme.primary
                )
            }
        }
    }
}
```


## Casos de Uso Actualizados

### SaveReadingUseCase (Actualizado)

```kotlin
class SaveReadingUseCase @Inject constructor(
    private val repository: ReadingHistoryRepository
) {
    suspend operator fun invoke(
        reading: TarotReading,
        interpretation: Interpretation,
        consultantName: String = "Lectura personal" // v1.2.0: Valor por defecto
    ): Result<Long> {
        val readingHistory = ReadingHistory(
            id = 0,
            timestamp = System.currentTimeMillis(),
            consultantName = consultantName, // Usa valor proporcionado o por defecto
            spreadType = reading.spreadType,
            question = reading.question,
            drawnCards = reading.drawnCards,
            interpretation = interpretation,
            notes = emptyList()
        )

        return repository.saveReading(readingHistory)
    }
}
```

## Riesgos y Mitigaciones

### Riesgo 1: Imagen de dorso no disponible

**Impacto**: Alto - Bloquea desarrollo de ManualLoadScreen

**Mitigación**:
- Usar placeholder temporal (color sólido + texto "Dorso")
- Buscar imagen de dominio público en Wikimedia Commons
- Crear imagen simple con herramienta de diseño (Figma/Canva)

### Riesgo 2: Performance con múltiples imágenes en CardSelectorScreen

**Impacto**: Medio - Grid con 78 imágenes puede ser lento

**Mitigación**:
- Usar `AsyncImage` de Coil (caché automática)
- Implementar paginación o scroll infinito si es necesario
- Reducir calidad de imágenes en selector (thumbnail)

### Riesgo 3: Inconsistencia UX entre ReadingScreen y ManualLoadScreen

**Impacto**: Alto - Confunde al usuario

**Mitigación**:
- Reutilizar componentes existentes (HorizontalCardsLayout, etc.)
- Aplicar mismos estilos y spacing
- Testing de usabilidad antes de release

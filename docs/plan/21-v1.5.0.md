# Plan de Implementación v1.5.0 - Modernización de HomeScreen

> **Versión**: 1.5.0
> **Estado**: 🆕 Planificación
> **Fecha**: 2026-05-15
> **Requisito**: RF-22

---

## Contexto

La HomeScreen actual tiene un diseño funcional pero genérico que no aprovecha al máximo las capacidades visuales de Material Design 3 ni refleja la identidad de marca de la aplicación. Esta versión se enfoca en modernizar la pantalla principal para mejorar la primera impresión y la experiencia de usuario.

### Problemas Identificados

1. **TopAppBar redundante**: El título "Arcana" ocupa espacio sin aportar valor
2. **Botones genéricos**: No comunican visualmente la función de cada opción
3. **Falta de branding**: El logo solo aparece como texto plano
4. **Márgenes del sistema**: No respeta status bar ni navigation bar apropiadamente
5. **Botón sin función**: "Configuración" visible pero no implementado

---

## Decisiones Arquitecturales

### 1. Componentes Reutilizables

#### HomeOptionCard
Componente modular y reutilizable para las opciones de la home.

**Razones**:
- Consistencia visual entre todas las opciones
- Facilita mantenimiento y cambios futuros
- Permite variantes (primaria/secundaria)
- Reduce duplicación de código

**Estructura**:
```kotlin
@Composable
fun HomeOptionCard(
    icon: ImageVector,
    title: String,
    description: String,
    onClick: () -> Unit,
    modifier: Modifier = Modifier,
    isPrimary: Boolean = false
)
```

**Ubicación**:
- Opción 1 (recomendada): `presentation/screens/components/HomeOptionCard.kt`
- Opción 2 (inline): Directamente en `HomeScreen.kt` como función privada

**Decisión**: Opción 1 - Componente separado
- Facilita testing aislado
- Mejor organización del código
- Potencial reutilización en futuras pantallas

### 2. Gestión de Márgenes del Sistema

#### WindowInsets vs Modifiers

**Opciones evaluadas**:

**A. Modifier.systemBarsPadding()**
```kotlin
Scaffold(
    modifier = Modifier.systemBarsPadding()
) { ... }
```
✅ Simple y directo
✅ Automático para status + navigation
❌ Menos control granular

**B. Modifier.statusBarsPadding() + navigationBarsPadding()**
```kotlin
Column(
    modifier = Modifier
        .statusBarsPadding()
        .navigationBarsPadding()
) { ... }
```
✅ Control granular por eje
✅ Permite composición flexible
❌ Más verboso

**C. WindowInsets API**
```kotlin
Scaffold(
    contentWindowInsets = WindowInsets.systemBars
) { ... }
```
✅ Más moderno (Android 11+)
✅ Mejor para edge-to-edge
❌ Complejidad adicional para caso simple

**Decisión**: Opción B
- Máximo control granular
- Compatible con minSdk 24
- Claridad en la intención
- Permite ajustes específicos si es necesario

---

### 3. Iconografía

#### Material Icons vs Custom Icons

**Opciones**:

**A. Material Icons Default**
```toml
# Ya incluido en Compose
compose-material3
```
✅ Sin dependencias extra
✅ 1000+ iconos disponibles
❌ Algunos iconos genéricos

**B. Material Icons Extended**
```toml
material-icons-extended = "1.7.6"
```
✅ 5000+ iconos (incluye `History`, `AutoStories`)
✅ Mejor semántica visual
❌ +500KB al APK

**C. Custom Icons**
```kotlin
// Iconos vectoriales personalizados
R.drawable.ic_tarot_deck
```
✅ Personalización total
✅ Branding único
❌ Requiere diseño manual o compra

**Decisión**: Opción A + Mejores iconos disponibles
- Usar `Icons.Default` como base
- Reemplazar solo iconos confusos:
  - Enciclopedia: `LibraryBooks` (mejor que `Face`)
  - Historial: `Assignment` o `List` (mejor que `Info`)
- **NO agregar** `material-icons-extended` (peso excesivo para 2 iconos)

**Mapeo final de iconos**:
| Función | Icono | Justificación |
|---------|-------|---------------|
| Nueva Lectura | `Icons.Default.Star` | Místico, especial, acción principal |
| Carga Manual | `Icons.Default.AddCircle` | Acción de agregar/cargar |
| Enciclopedia | `Icons.Default.LibraryBooks` | Libro/biblioteca (más apropiado que Face) |
| Historial | `Icons.Default.Assignment` | Lista/registro histórico |

---

### 4. Jerarquía Visual

#### Card Primaria vs Secundarias

**Estrategia de diferenciación**:

**Card Primaria (Nueva Lectura)**:
- `containerColor = primaryContainer`
- `contentColor = onPrimaryContainer`
- Acción core de la app
- Mayor contraste visual

**Cards Secundarias**:
- `containerColor = surface` (default)
- `contentColor = onSurface` (default)
- Acciones complementarias
- Diseño más sutil

**Elevation**:
- Todas las cards: `2.dp`
- Consistente con Material Design 3
- Sutil pero perceptible

**Ripple Effect**:
- Automático con `Modifier.clickable`
- Feedback táctil inmediato
- Color según containerColor

### 5. Logo - Tratamiento de Imagen

#### Especificaciones

**Imagen existente**: `nombre_solo.png`
- Formato: PNG
- Ubicación: `drawable/nombre_solo.png`
- Contenido: Logo "Arcana" + gráfico

**Implementación**:
```kotlin
Image(
    painter = painterResource(id = R.drawable.nombre_solo),
    contentDescription = "Arcana - Logo de la aplicación",
    modifier = Modifier
        .fillMaxWidth(0.6f)        // 60% del ancho
        .padding(vertical = 32.dp), // Espacio arriba/abajo
    contentScale = ContentScale.Fit  // Mantener proporciones
)
```

**Razones del tamaño (60%)**:
- Balance entre impacto visual y espacio para cards
- No domina la pantalla pero es prominente
- Adaptable a diferentes tamaños de pantalla
- Probado en splash screen con buenos resultados

**Alternativa si imagen es muy grande**: `fillMaxWidth(0.5f)` (50%)

### Archivos a Crear

#### 1. HomeOptionCard.kt
**Ubicación**: `presentation/screens/components/HomeOptionCard.kt`

**Contenido**:
```kotlin
package com.waveapp.tarotai.presentation.screens.components

import androidx.compose.foundation.clickable
import androidx.compose.foundation.layout.*
import androidx.compose.material3.*
import androidx.compose.runtime.Composable
import androidx.compose.ui.Alignment
import androidx.compose.ui.Modifier
import androidx.compose.ui.graphics.vector.ImageVector
import androidx.compose.ui.unit.dp

/**
 * Card clickeable para opciones de la HomeScreen.
 *
 * Representa una acción disponible con:
 * - Icono visual (izquierda)
 * - Título y descripción (derecha)
 * - Ripple effect al tocar
 *
 * Soporta dos variantes:
 * - **Primaria**: Acción principal destacada (primaryContainer)
 * - **Secundaria**: Acciones complementarias (surface)
 *
 * @param icon Icono representativo de la acción
 * @param title Nombre corto de la acción
 * @param description Texto explicativo de qué hace la acción
 * @param onClick Lambda ejecutada al tocar la card
 * @param modifier Modificador opcional para la card
 * @param isPrimary Si true, usa colores de primaryContainer (destacado)
 */
@Composable
fun HomeOptionCard(
    icon: ImageVector,
    title: String,
    description: String,
    onClick: () -> Unit,
    modifier: Modifier = Modifier,
    isPrimary: Boolean = false
) {
    Card(
        modifier = modifier
            .fillMaxWidth()
            .clickable(onClick = onClick),
        elevation = CardDefaults.cardElevation(
            defaultElevation = 2.dp
        ),
        colors = if (isPrimary) {
            CardDefaults.cardColors(
                containerColor = MaterialTheme.colorScheme.primaryContainer
            )
        } else {
            CardDefaults.cardColors()
        }
    ) {
        Row(
            modifier = Modifier
                .fillMaxWidth()
                .padding(16.dp),
            verticalAlignment = Alignment.CenterVertically
        ) {
            // Icono
            Icon(
                imageVector = icon,
                contentDescription = null,
                modifier = Modifier
                    .size(48.dp)
                    .padding(end = 16.dp),
                tint = if (isPrimary) {
                    MaterialTheme.colorScheme.onPrimaryContainer
                } else {
                    MaterialTheme.colorScheme.primary
                }
            )

            // Título y descripción
            Column(
                modifier = Modifier.weight(1f)
            ) {
                Text(
                    text = title,
                    style = MaterialTheme.typography.titleMedium,
                    color = if (isPrimary) {
                        MaterialTheme.colorScheme.onPrimaryContainer
                    } else {
                        MaterialTheme.colorScheme.onSurface
                    }
                )

                Spacer(modifier = Modifier.height(4.dp))

                Text(
                    text = description,
                    style = MaterialTheme.typography.bodySmall,
                    color = if (isPrimary) {
                        MaterialTheme.colorScheme.onPrimaryContainer.copy(alpha = 0.7f)
                    } else {
                        MaterialTheme.colorScheme.onSurfaceVariant
                    }
                )
            }
        }
    }
}
```

**Razones del diseño**:
- Documentación clara para mantenimiento futuro
- Parámetros semánticos (no técnicos)
- Composición simple (Row + Column)
- Colores dinámicos según tema
- Accesibilidad por defecto (Ripple, contrast)

## Riesgos y Mitigaciones

### 1. Logo muy grande/pequeño

**Riesgo**: El logo `nombre_solo.png` puede verse desproporcionado en algunos dispositivos.

**Mitigación**:
- Usar `fillMaxWidth(0.6f)` como valor base
- Si muy grande: reducir a `0.5f`
- Si muy pequeño: aumentar a `0.7f`
- Testing en diferentes tamaños de pantalla

---

### 2. Cards muy juntas

**Riesgo**: En pantallas pequeñas, las 4 cards pueden verse apretadas.

**Mitigación**:
- Usar `verticalScroll` para permitir scroll
- Espaciado de 12dp entre cards
- Padding horizontal de 24dp para respirar
- Testing en 4.7" y 6.5"+

---

### 3. Iconos confusos

**Riesgo**: `LibraryBooks` o `Assignment` pueden no ser intuitivos para todos los usuarios.

**Mitigación**:
- Descripción clara en cada card
- Testing con usuarios reales
- Posibilidad de cambiar a otros iconos si es necesario:
  - Enciclopedia: `MenuBook`, `Book`
  - Historial: `List`, `Receipt`

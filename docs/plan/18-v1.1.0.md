# 18. Cambios de v1.1.0

## Resumen de v1.1.0

**v1.1.0** agrega dos funcionalidades profesionales para tarotistas:

1. **Historial de Lecturas Guardadas** (RF-11)
2. **Carga Manual de Tiradas** (RF-12)

Ambas funcionalidades comparten el flujo inicial con tiradas automáticas para máxima coherencia UX.

## Actualización de QuestionScreen (Reutilización)

**QuestionScreen** ahora acepta parámetro `isManualLoad`:

```kotlin
@Composable
fun QuestionScreen(
    spreadType: SpreadType,
    isManualLoad: Boolean = false,    // 🆕 Nuevo parámetro
    onContinue: (question: String, consultantName: String?) -> Unit,
    onNavigateBack: () -> Unit
)
```

**Comportamiento**:
- **isManualLoad = false**: Toggle consultante opcional (OFF por defecto)
- **isManualLoad = true**: Toggle forzado a ON, consultante obligatorio

## Decisiones de Arquitectura

### 1. Reutilización de Pantallas ✅

**Decisión**: Reutilizar `SpreadTypeScreen` y `QuestionScreen` para ambos flujos (automático y manual).

**Razones**:
- Coherencia UX (usuario aprende flujo una vez)
- Menos código duplicado
- Mantenibilidad futura
- Preparado para RF-11 (consultante desde inicio)

### 2. Captura de Consultante

**Decisión**: Capturar consultante en `QuestionScreen` (opcional en automático, obligatorio en manual).

**Alternativa rechazada**: Solo pedir consultante al guardar.

**Razones**:
- Contexto fresco al inicio
- Mejor UX para usuarios profesionales
- Fallback disponible (diálogo al guardar si no se llenó)

### 3. Filtros Excluyentes en CardSelector

**Decisión**: Solo un filtro activo a la vez (radio buttons).

**Alternativa rechazada**: Filtros múltiples (checkboxes).

**Razones**:
- Usuario busca carta específica (no explora)
- Simplifica UI
- Diferencia clara con EncyclopediaScreen

### 4. Autosave de Notas

**Decisión**: Guardar automáticamente después de 2 segundos sin actividad.

**Alternativa rechazada**: Botón "Guardar" explícito.

**Razones**:
- Mejor UX (menos fricción)
- Estándar en apps modernas
- Indicador visual de guardado

# Cambios de v1.3.0 - Reconocimiento de Voz

> **Versión**: 1.3.0
> **Estado**: ⏳ Planificación
> **Fecha**: 2026-05-11

---

## Contexto

Esta versión agrega funcionalidad de reconocimiento de voz (Speech-to-Text) en `QuestionScreen`, permitiendo a los usuarios dictar sus preguntas en lugar de escribirlas, mejorando significativamente la experiencia de usuario especialmente para preguntas largas o reflexivas.

---

## Decisiones Arquitecturales

### 1. API de Reconocimiento de Voz

**Decisión**: Usar `SpeechRecognizer` API nativa de Android

**Alternativas Consideradas**:
- ❌ Google Cloud Speech-to-Text API: Requiere pago y configuración de credenciales
- ❌ Bibliotecas de terceros (Vosk, Mozilla DeepSpeech): Agregan peso a la APK
- ✅ **SpeechRecognizer (nativo)**: Gratuito, integrado, no requiere dependencias

**Justificación**:
- API nativa de Android (disponible desde API 8)
- Gratuita y sin configuración adicional
- Usa el motor de Google integrado en el dispositivo
- Soporte offline en dispositivos Android 12+ si tienen el modelo descargado
- Fallback automático a reconocimiento en la nube si no hay soporte offline
- No agrega peso a la APK

### 2. Gestión de Estado

**Decisión**: Agregar estado de reconocimiento de voz en `QuestionViewModel`

**Estructura**:
```kotlin
sealed class SpeechRecognitionState {
    object Idle : SpeechRecognitionState()
    object Listening : SpeechRecognitionState()
    object Processing : SpeechRecognitionState()
    data class Error(val message: String) : SpeechRecognitionState()
}
```

**Justificación**:
- Sigue el patrón MVVM existente
- Separación clara de responsabilidades (ViewModel maneja lógica, Screen maneja UI)
- Estados explícitos permiten UI reactiva

### 3. Manejo de Permisos

**Decisión**: Solicitar permiso en tiempo de ejecución usando `rememberLauncherForActivityResult`

**Patrón**:
```kotlin
val permissionLauncher = rememberLauncherForActivityResult(
    ActivityResultContracts.RequestPermission()
) { isGranted ->
    if (isGranted) {
        viewModel.startListening()
    } else {
        // Mostrar mensaje explicativo
    }
}
```

**Justificación**:
- Patrón moderno de Jetpack Compose para permisos
- No requiere `Fragment` o `Activity`
- Manejo explícito de aceptación/rechazo

### 4. Idioma de Reconocimiento

**Decisión**: Configurar reconocimiento en español (`es-ES`)

**Configuración**:
```kotlin
putExtra(RecognizerIntent.EXTRA_LANGUAGE, "es-ES")
putExtra(RecognizerIntent.EXTRA_LANGUAGE_MODEL, RecognizerIntent.LANGUAGE_MODEL_FREE_FORM)
```

**Justificación**:
- La app está en español
- Mejor precisión con modelo de idioma correcto
- `FREE_FORM` permite lenguaje natural (vs `WEB_SEARCH` que espera queries)

### 5. Resultados Parciales

**Decisión**: Habilitar resultados parciales en tiempo real

**Configuración**:
```kotlin
putExtra(RecognizerIntent.EXTRA_PARTIAL_RESULTS, true)
```

**Justificación**:
- Feedback inmediato al usuario
- Puede ver su pregunta formándose en tiempo real
- Mejora la sensación de "naturalidad" del reconocimiento

### 6. Integración con UI Existente

**Decisión**: Agregar icono de micrófono como `trailingIcon` del `OutlinedTextField`

**Ubicación**:
```kotlin
OutlinedTextField(
    value = question,
    onValueChange = { question = it },
    label = { Text("Tu pregunta") },
    trailingIcon = {
        IconButton(onClick = { /* lógica de voz */ }) {
            Icon(Icons.Default.Mic, ...)
        }
    }
)
```

**Justificación**:
- Ubicación estándar para acciones secundarias de input
- No interfiere con el teclado virtual
- Visualmente claro y accesible
- Similar a apps como WhatsApp, Google Messages

### 7. Feedback Visual Durante Reconocimiento

**Decisión**: Cambiar color y escala del icono según estado

**Estados visuales**:
- **Idle**: Icono gris/primario, tamaño normal
- **Listening**: Icono rojo, escala 1.2x (efecto pulsante)
- **Processing**: CircularProgressIndicator pequeño

**Justificación**:
- Feedback inmediato y claro
- Rojo universalmente asociado con "grabando"
- Animación sutil sin distraer

## Flujo de Datos

### Inicialización
```
QuestionScreen (Composable)
    ↓
LaunchedEffect { viewModel.initSpeechRecognizer(context) }
    ↓
QuestionViewModel.initSpeechRecognizer()
    ↓
SpeechRecognizer.createSpeechRecognizer(context)
    ↓
setRecognitionListener(...)
```

### Reconocimiento de Voz
```
Usuario toca icono micrófono
    ↓
Verificar permiso RECORD_AUDIO
    ↓ (si no tiene)
    permissionLauncher.launch(RECORD_AUDIO)
    ↓ (si concede)
viewModel.startListening()
    ↓
SpeechRecognizer.startListening(intent)
    ↓
onBeginningOfSpeech() → _speechState = Listening
    ↓
onPartialResults() → _question.value = partialText (tiempo real)
    ↓
onResults() → _question.value = finalText
    ↓
_speechState = Idle
```

### Manejo de Errores
```
onError(errorCode)
    ↓
Traducir errorCode a mensaje en español
    ↓
_speechState = Error(message)
    ↓
Screen muestra error en Text rojo
```

## Manejo de Errores

### Códigos de Error y Mensajes

| Error Code | Constant | Mensaje en Español |
|------------|----------|-------------------|
| 1 | `ERROR_NETWORK_TIMEOUT` | "Tiempo de espera agotado" |
| 2 | `ERROR_NETWORK` | "Error de conexión" |
| 3 | `ERROR_AUDIO` | "Error de audio" |
| 4 | `ERROR_SERVER` | "Error del servidor" |
| 5 | `ERROR_CLIENT` | "Error del cliente" |
| 6 | `ERROR_SPEECH_TIMEOUT` | "No se detectó voz" |
| 7 | `ERROR_NO_MATCH` | "No se entendió el audio" |
| 8 | `ERROR_RECOGNIZER_BUSY` | "Reconocedor ocupado" |
| 9 | `ERROR_INSUFFICIENT_PERMISSIONS` | "Permiso de micrófono denegado" |

### Estrategia de Recuperación

- **ERROR_NETWORK**: Sugerir al usuario que verifique su conexión
- **ERROR_NO_MATCH / ERROR_SPEECH_TIMEOUT**: Sugerir hablar más claro o más cerca del micrófono
- **ERROR_INSUFFICIENT_PERMISSIONS**: Explicar que el permiso es necesario para dictar
- **Otros errores**: Botón "Reintentar" visible

## Performance y Optimización

### Lifecycle de SpeechRecognizer

**Importante**: El `SpeechRecognizer` debe destruirse correctamente

```kotlin
override fun onCleared() {
    super.onCleared()
    speechRecognizer?.destroy()
}
```

**Justificación**:
- Evitar memory leaks
- Liberar recursos del sistema (micrófono)
- Buena práctica de Android

### Timeout Automático

**Configuración**: Android maneja timeout automáticamente (5-10 segundos sin voz)

**Ventaja**: No requiere lógica adicional de timeout

## Consideraciones de Seguridad

### Permiso de Micrófono

**Nivel de permiso**: `dangerous` (requiere solicitud en tiempo de ejecución)

**Best Practices**:
- ✅ Solo solicitar cuando el usuario toca el botón de micrófono
- ✅ Explicar para qué se usará (dictar pregunta)
- ✅ No bloquear la app si se niega (teclado sigue disponible)
- ✅ No usar el micrófono para nada más

### Privacidad

- ✅ El audio NO se guarda en ningún lado
- ✅ El texto reconocido solo se usa para la pregunta
- ✅ Reconocimiento lo hace Google (o motor del dispositivo)
- ✅ No enviamos audio a nuestro backend

## Riesgos y Mitigaciones

### Riesgo 1: Dispositivo sin soporte de reconocimiento

**Probabilidad**: Baja (mayoría de dispositivos Android tienen Google App)

**Mitigación**:
```kotlin
if (!SpeechRecognizer.isRecognitionAvailable(context)) {
    // Ocultar botón de micrófono
    // o mostrar mensaje "No disponible en este dispositivo"
}
```

### Riesgo 2: Baja precisión en español

**Probabilidad**: Media (depende de la voz del usuario y ruido ambiente)

**Mitigación**:
- Usuario puede editar texto después de reconocimiento
- Mostrar claramente que es una "asistencia", no reemplazo del teclado

### Riesgo 3: Usuario niega permiso de micrófono

**Probabilidad**: Media

**Mitigación**:
- No bloquear la funcionalidad
- Teclado sigue funcionando normalmente
- Mensaje claro explicando que puede otorgar permiso más tarde en Configuración
